# üì¶ SUMMARY  
# This set of commands navigates to the /opt directory, downloads a compressed archive of Nexus Repository Manager, lists files in the directory, and extracts the downloaded archive.  
# Language/Platform: Bash/Linux shell commands  
# What you'll learn: How to change directories, download files from the internet, list directory contents, and extract tar.gz archives  
# Outcome: The Nexus Repository Manager archive is downloaded and extracted into the /opt directory  
# Why this matters: Setting up Nexus repository allows managing and hosting software artifacts, essential in development pipelines  

# ------------------------------  
# üß© SECTION TITLE: Change directory to /opt  
# ------------------------------  
cd /opt/  
# ‚úÖ What it does: Changes the current directory to /opt, a common location for optional software installations  
# üß† Why it matters: Ensures you download and extract files in the right system directory for software  

# ------------------------------  
# üß© SECTION TITLE: Download Nexus Repository archive  
# ------------------------------  
wget https://download.sonatype.com/nexus/3/nexus-unix-x86-64-3.79.0-09.tar.gz  
# ‚úÖ What it does: Downloads the Nexus Repository Manager tarball from Sonatype's official URL to the current directory  
# üß† Why it matters: Gets the installation package needed to set up Nexus locally  

# ------------------------------  
# üß© SECTION TITLE: List contents of current directory  
# ------------------------------  
ls  
# ‚úÖ What it does: Lists all files and folders in the current directory (/opt)  
# üß† Why it matters: Verifies that the downloaded file is present before extracting  

# ------------------------------  
# üß© SECTION TITLE: Extract the Nexus archive  
# ------------------------------  
tar -zxvf nexus-unix-x86-64-3.79.0-09.tar.gz  
# ‚úÖ What it does: Extracts the .tar.gz archive into the current directory, creating the Nexus folder structure  
# üß† Why it matters: Unpacks the software so it can be installed or run from the extracted files  

# ------------------------------  
# ‚ñ∂Ô∏è HOW TO RUN / EXECUTE THESE COMMANDS  
# ------------------------------  
# Bash/Linux shell:  
cd /opt/                             # ‚úÖ Change directory to /opt where software will be installed  
                                    # üß† Necessary to keep system organized with standard locations for optional software

                                    # ‚úÖ Download Nexus installation package  
                                    # üß† Fetches the software needed for repository management

                                    # üß† Helps verify the environment before extraction  
tar -zxwf nexus-unix-x86-64-3.79.0-09.tar.gz  
                                    # ‚úÖ Extracts the archive contents into /opt  
                                    # üß† Makes the software ready for configuration or execution

# ------------------------------  
# ‚úÖ NOTES / GOTCHAS / TIPS  
# ------------------------------  
# - Make sure you have permissions to write in /opt (may need `sudo` before commands).  
# - `wget` requires network access and that the URL is reachable.  
# - The `tar` flags: `-z` decompresses gzip, `-x` extracts, `-w` writes files, `-f` specifies the archive file.
